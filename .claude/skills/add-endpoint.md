# Add a new FastAPI endpoint

Use this skill when extending the backend API. Following the conventions consistently
keeps the codebase coherent.

## Steps

### 1. Define the Pydantic schemas

Add request + response schemas to `backend/app/schemas.py`. One schema per shape — don't
overload. Example:

```python
class FooCreate(BaseModel):
    name: str
    description: str | None = None

class FooOut(BaseModel):
    model_config = ConfigDict(from_attributes=True)
    id: int
    name: str
    description: str | None = None
```

### 2. Add the endpoint to `backend/app/main.py`

```python
from .schemas import FooCreate, FooOut

@app.post("/api/foos", response_model=FooOut)
def create_foo(
    req: FooCreate,
    claims: dict = Depends(require_auth),
    creds: HTTPAuthorizationCredentials = Security(_bearer_scheme),
    db: Session = Depends(get_db),
) -> models.Foo:
    """One-line description of what this endpoint does and when to call it."""
    foo = models.Foo(name=req.name.strip(), description=req.description)
    db.add(foo)
    db.commit()
    db.refresh(foo)
    return foo
```

**Required dependencies:**
- `claims: dict = Depends(require_auth)` — 401s without a valid Entra token
- `db: Session = Depends(get_db)` — request-scoped session (sandbox-aware!)
- `creds` if you need the raw token (e.g. to call Graph /me)

### 3. Import the schema at the top of `main.py`

Add to the existing import block, NOT a separate one.

### 4. Add to the schemas import in `main.py`

```python
from .schemas import (
    # ... existing ...
    FooCreate, FooOut,
)
```

### 5. Add the frontend API helper

In `frontend/src/api.ts`:

```typescript
export interface FooCreatePayload {
  name: string;
  description?: string | null;
}

export interface Foo {
  id: number;
  name: string;
  description: string | null;
}

export async function createFoo(payload: FooCreatePayload): Promise<Foo> {
  const r = await authFetch(`${BASE}/foos`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(payload),
  });
  if (!r.ok) throw new Error(`Create foo failed: ${r.status}`);
  return r.json();
}
```

Use `authFetch`, never bare `fetch` — it auto-attaches the Bearer token + retries on 401.

### 6. Type-check

```powershell
cd frontend
npx tsc -b
```

### 7. Test in Test Mode

Toggle Test Mode → exercise the new endpoint from the UI → verify it writes to sandbox.

## Conventions to follow

- **Never `db.add_all` followed by `db.commit()` without `db.refresh()` if you return the object** — the response won't have the autogenerated PK.
- **Lowercase emails, uppercase t-shirt sizes** when storing — see `create_team_manual` in `main.py` for the pattern.
- **Empty string → None for nullable text fields**: `(req.field or "").strip() or None`.
- **Soft duplicate checks** use case-insensitive: `db.query(Foo).filter(Foo.name.ilike(name)).first()`.
- **Status code conventions**: 201 implicit on POST (FastAPI), 404 for not-found, 409 for duplicates, 403 for permission denied, 400 for bad input, 422 only for Pydantic validation errors (automatic).
- **Audit log** any destructive / important mutation via `_audit_team_edit` in `main.py` (or follow that pattern for other tables).
